# Indonesia Tourism Ontology (Bali · NTB · NTT)

An OWL 2 DL ontology covering tourism in three Indonesian provinces — **Bali**,
**West Nusa Tenggara (NTB)**, and **East Nusa Tenggara (NTT)** — populated
automatically from DBpedia, enriched with manually curated data, verified for
consistency with the **HermiT** reasoner, and analysed with **knowledge graph
embeddings** for link prediction.

The TBox (classes, properties, axioms) is hand-modeled in Protégé. The ABox
(individuals and their relations) is generated by a modular Python pipeline and
exposed as an interactive Jupyter notebook.

## Setup

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

You also need a working **Java** runtime on `PATH` (HermiT is a `.jar`).

## Usage

### Notebook (recommended)

Open `pipeline.ipynb` in Jupyter and run cells top to bottom:

| Cell | What it does |
|---|---|
| 1 | Imports + logging setup |
| 2 | Reset `data.owl` → populate from DBpedia |
| 3 | Enrich + save + HermiT consistency check |
| 4 | Train DistMult embeddings (t-SNE plot) |
| 5 | Manual link prediction queries |
| 6 | Interactive HTML network graph |

### Command line

```bash
# Full pipeline: reset → populate → enrich → save → reason
python main.py

# Render ontology as interactive HTML graph
python visualize.py
open ontology_graph.html
```

`main.py` is idempotent — it wipes `data.owl` to a clean skeleton on every run.

## Pipeline detail

### Populate (DBpedia SPARQL)

Queries DBpedia for each class and adds `rdf:type` + `locatedIn` triples.

| Class | Coverage |
|---|---|
| Province | 3 (Bali, NTB, NTT) |
| Island | ~27 |
| City / Regency | ~39 |
| Beach | ~8 |
| Park | 6 |
| Volcano | 4 |
| Museum | ~11 |
| Temple | ~5 |
| Festival | ~2 (DBpedia) + 6 manual |
| ReligiousCeremony | ~15 |
| Hotel | ~5 |

### Enrich (17 steps)

1. Country backbone — Province → `locatedInCountry` → Indonesia  
2. Bali Island — manual fix (DBpedia types Bali as Province, not Island)  
3. Island → Province links via `isPartOf` / `wikiPageWikiLink`  
4. Location links — `wikiPageWikiLink` → `locatedIn / locatedInIsland / locatedInCity`  
5. Activity individuals — Surfing, Diving, Hiking, Cultural_Tour, …  
6. Activity links — DBpedia categories + `wikiPageWikiLink` → `hasActivity`  
7. Activity fallbacks — default activities for entities DBpedia didn't cover  
8. Attraction hubs — capital city → `hasTouristAttraction` → attractions in province  
9. Accommodation hubs — capital city → `hasAccommodation` → hotels in province  
10. Visitor counts — `numberOfVisitors` from DBpedia (parks)  
11. Ratings — manually curated `hasRating` (xsd:decimal, scale 1–5)  
12. Entry fees — manually curated `hasEntryFee` (xsd:boolean)  
13. Established years — manually curated `establishedYear` for parks  
14. Transportation — airports, ports, terminals per province  
15. Festivals — NTB / NTT festivals (DBpedia coverage too sparse)  
16. Traditional dances — Kecak, Legong, Barong, Tari Oncer, Hegong, …  
17. Traditional houses — Bale Banjar, Mbaru Niang, Rumah Adat Sumba, …

## Graph embeddings & link prediction

`graph_embedding.py` trains a **DistMult** model (64-dim, 400 epochs) on all
object-property triples using PyKEEN. After training:

- **t-SNE plot** — entities projected to 2D, coloured by class (`embedding_clusters.png`)
- **Link prediction** — given `(head, relation, ?)`, the model ranks candidate
  tails filtered to a target class

The model learns purely from graph structure — it has no external world knowledge.
Useful predictions are for entities with **missing** links, not for links already
in the training data.

## Known limitations

- DBpedia has very sparse data for NTB ceremonies and festivals — `Bau_Nyale` and
  `Perang_Topat` don't exist as DBpedia resources.
- `Pasola` and `Caci` (NTT) are categorised as martial arts in DBpedia; pulled
  explicitly via `VALUES`.
- HermiT infers Provinces as City via `locatedInCity range = City` — this is a
  schema design issue, not a pipeline bug.
- With ~300 individuals, embedding quality is limited; predictions are best treated
  as structural interpolation rather than semantic reasoning.
